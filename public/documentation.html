myFlix

myFlix will provide users with access to information about different

movies, directors, and genres. Users will be able to sign up, update their

personal information, and create a list of their favorite movies.

Business Logic URL HTTP Method Query parameters Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None None A JSON object holding data for all movies
Return data for a single movie by title to the user /movies/[title] GET :title None A JSON object holding data for the selected movie containing description, genre, director and img URL. Example: 
                    
                      {
                        Title: '', 
                        Description: '',
                        Year: '', 
                        Genre: {
                        name: '',
                        description: ''
                      } 
                       Director: {
                       name: '', 
                       birth: '',
                       death: ''
                       bio: ''
                     },
                      Actors: {
        
                     }
                      ImagePath: ''
                      Featured: true
                     }
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/genres/[genre] GET :genre None A JSON object holding data about the selected genre. Example: 
                    
                    { 
                      name: 'Religious', 
                      description: '', 
                    }
Return data about a director (bio, birth year, death year) by name /movies/directors/[name] GET :name None A JSON object holding data about the selected director. Example: 
                    
                     { 
                       name: 'Franco Zeffirelli', 
                       birth: '1968', 
                       death: '2019',
                       bio: ''
                     }
Allow new users to register /users POST None A JSON object holding data about the user to add. Example: 
                     
                       { 
                         "username": "johnt", 
                         "email": "johnt@gmail.com", 
                         "password": "hellothere"
                         "birthday": "11/06/1990", 
                         "favorites": [
                         "Jesus of Nazareth", "Peter and Paul"
                       ] 
                     }
A JSON object holding data about added user including ID. Example: 
                      
                        { 
                          id: '1',
                          username: 'johnt', 
                          email: 'johnt@gmail.com', 
                          password: 'hellothere'
                          birthday: '11/06/1990',
                          favorites: [
                         'Jesus of Nazareth', 'Peter and Paul'
                            ] 
                        }
Allow users to update their user info (username) /users/[username] PUT :username A JSON object holding the new username. Example: 
                     
                       { 
                         "username": "johnts"
                       }
A JSON object with the updated user profile. Example: 
                      
                         { 
                          id: '1',
                          username: 'johnts', 
                          email: 'johnts@gmail.com', 
                          password: 'hellothere'
                          birthday: '11/06/1990', 
                          favorites: [
                            'Jesus of Nazareth', 'Peter and Paul'
                            ] 
                         }
Allow users to add a movie to their list of favorites /users/[username]/[movie] POST :username, :movie None A text message indicating the movie was added
Allow users to remove a movie from their list of favorites /users/[username]/[movie] DELETE :username, :movie None A text message indicating the movie was deleted
Allow existing users to remove their account /users/[username] DELETE :username None A message indicating the user was removed
